Descripción General § 1

ITF9000 es un servicio transaccional bancario desarrollado en ILE RPG para IBM AS/400 (IBM i). Su propósito principal es procesar operaciones de cargo y abono sobre cuentas bancarias dentro del sistema de fondos mutuos. Opera como un backend service invocado por canales externos (IBank, CCA, RBN, etc.) y ejecuta validaciones, movimientos contables y notificaciones de forma atómica.
Modificación IS-907 / Cambio 8109: Se eliminó la llamada al programa ETF5000 que existía en la versión anterior del servicio.

Flujo de Ejecución — Paso a Paso § 4

1
Inicialización — *inzsr
Recibe parámetros: port, rformat, rbuffer. Lee estado del sistema desde área de datos (DTAARA): modo Online/Offline → *IN69. Lee CNTRLCNT para obtener RUNDTE (fecha contable base). Define todas las claves compuestas (KLIST) y alias de variables.
2
Limpieza y decodificación del buffer
Se limpian las estructuras ITF900001 y perrormsg. Se valida que rformat = 'ITF900001', de lo contrario se registra error 0953. Se carga el buffer de entrada en la estructura de datos ITF900001.
3
Autorización — sr_autoriza
Verifica que la combinación ih01userid + ih01progrm exista en la tabla CNTIN con código 700. Si no existe, activa $autoriza = 'N' y se ejecuta sr_autherror (error 2175).
4
Parametrización contable — @tdopecon
Consulta TDOPECON dos veces: primero con tipo 'DB' (débito) y luego 'CR' (crédito) usando ie01sisori como clave. Carga número de lote, código de transacción, cuenta, sucursal y glosa por defecto para cada lado del asiento.
5
Análisis de fecha contable — @anzfc
Determina la fecha contable final considerando: RUNDTE, hora actual vs hora de corte (Post-14), canal externo (H01TIMSYS), y calendario de días hábiles. Activa *IN65 si la operación es Post-14 (fecha contable = día siguiente hábil).
6
Despacho por código de operación — ih01opecod
Bifurca la lógica: '0001' → @corrientes (transferencia entre cuentas corrientes), '0002' → @contables (transferencia con cuenta GL), cualquier otro valor → error 1039.
7
Procesamiento transaccional
El flujo interno de @corrientes y @contables ejecuta en orden: validaciones comunes → validación cuenta débito → validación cuenta crédito → validación sobregiro → retención de fondos → generación de folio → armado de glosas → escritura de asientos → escritura de cabecera → notificación.
8
Respuesta y auditoría
@senderror formatea errores con ERRORMSG si e1 > 0. @sbrsend carga el buffer de respuesta. sr_graba_log escribe registro completo en FFTRBALOG. Finalmente *inlr = *on termina el programa.
